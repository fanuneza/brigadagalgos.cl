# Success Dogs Content Collections — Design Spec

**Goal:** Migrate the 18 success-story dogs from a client-side `fetch('/casos/exito.json')` pipeline into an Astro 6 Content Collection, render all 18 cards at build time, and add a "Ver más" button that reveals cards 6 at a time.

**Date:** 2026-05-02

---

## 1. Current vs. Target Architecture

### Current
```
index.astro
  └─ <StoriesSection />            (no props)
       └─ <script> fetch('/casos/exito.json')
                   normalizeCase() → pick 6 random
                   createCard() → build DOM
                   initSharedGalleries()
```
Images served from `public/casos/exito/[id]/` as pre-processed webp variants (generated by `prepare-casos-site.mjs`).

### Target
```
index.astro (build time)
  └─ getCollection('success-dogs') → getImage() × 3 per photo
  └─ <StoriesSection dogs={dogs} />  (pre-built array)
       └─ 18 <article> pre-rendered (Astro)
       └─ <script> shuffle DOM → show 6 → "Ver más" reveals next 6
```
Images served from `src/assets/casos/exito/[id]/` — Astro generates hashed avif variants.

## 2. Content Model

**Collection name:** `success-dogs`  
**Location:** `src/content/success-dogs/[id].md`  
**Schema fields:**

| Field | Type | Notes |
|-------|------|-------|
| `name` | `string` | Display name (nombre_publico) |
| `story` | `string` | historia_breve_site — max ~240 chars |
| `gallery` | `z.array(image())` | Source images; 2–6 per dog |

No `sex`, `age`, `weight`, or `status` — stories only need name, text, and photos.

**Example entry (`src/content/success-dogs/leo.md`):**
```markdown
---
name: Leo
story: "Leo fue atropellado en Lampa y lo rescatamos cuando querían eutanasiarlo por falta de recursos. Pasó por dos cirugías por su fractura de espalda y hoy está adoptado; no camina, pero eso no le ha quitado las ganas de vivir bonito."
gallery:
  - ../../assets/casos/exito/leo/leo-01.jpg
  - ../../assets/casos/exito/leo/leo-02.jpg
  - ../../assets/casos/exito/leo/leo-03.jpg
  - ../../assets/casos/exito/leo/leo-04.jpg
---
```

**Image paths** are relative from `src/content/success-dogs/` → `../../assets/casos/exito/[id]/[file]`.

## 5. Build-Time Pipeline (`index.astro`)

```typescript
import { getCollection } from 'astro:content';
import { getImage } from 'astro:assets';
import type { SharedGalleryPhoto } from '../scripts/shared-gallery';

const rawDogs = await getCollection('success-dogs');

const dogs = await Promise.all(
  rawDogs.map(async (entry) => ({
    id: entry.id,
    name: entry.data.name,
    story: entry.data.story,
    photos: await Promise.all(
      entry.data.gallery.map(async (img): Promise<SharedGalleryPhoto> => ({
        card_mobile:  (await getImage({ src: img, width: 400,  format: 'avif' })).src,
        card_desktop: (await getImage({ src: img, width: 700,  format: 'avif' })).src,
        lightbox:     (await getImage({ src: img, width: 1200, format: 'avif' })).src,
      }))
    ),
  }))
);
```

No shuffle in `index.astro` — the DOM shuffle runs on the client at page load.

Pass `dogs` as a prop: `<StoriesSection dogs={dogs} />`.

## 6. `StoriesSection` Component Redesign

### Props
```typescript
interface StoryDog {
  id: string;
  name: string;
  story: string;
  photos: SharedGalleryPhoto[];
}

interface Props {
  dogs: StoryDog[];
}
```

### New imports (frontmatter)
`StoriesSection.astro` must add:
```
import SharedPhotoGallery from './SharedPhotoGallery.astro';
import type { SharedGalleryPhoto } from '../scripts/shared-gallery';
```

### Template changes
- **Remove:** skeleton `<div data-stories-skeleton>` (3 skeleton articles)
- **Remove:** `<p data-stories-status>` error message element
- **Remove:** the entire current `<script>` block (fetch, normalizeCase, createCard, loadStories)
- **Change:** `<div class="stories__grid" data-stories-list hidden>` → no `hidden`, render all dogs statically
- **Add:** "Ver más" button after the grid (hidden when total ≤ 6)
- **Keep:** `<SharedGalleryLightbox ariaLabel="Galería de historias" />`

### Static card markup (rendered per dog in Astro)
```astro
{dogs.map((dog) => (
  <article class="story-card" data-story-card>
    <SharedPhotoGallery name={dog.name} photos={dog.photos} />
    <div class="story-card__body">
      <p class="story-card__name">{dog.name}</p>
      <p class="story-card__quote">{dog.story}</p>
    </div>
  </article>
))}
```

### "Ver más" button (after the grid)
```astro
{dogs.length > 6 && (
  <div class="stories__ver-mas-wrap">
    <button type="button" class="btn btn--secondary" data-stories-ver-mas>
      Ver más historias
    </button>
  </div>
)}
```

### Replacement script
```typescript
import { initSharedGalleries, initSharedGalleryLightbox } from '../scripts/shared-gallery';

const PAGE_SIZE = 6;
const grid = document.querySelector<HTMLElement>('[data-stories-list]');
const button = document.querySelector<HTMLButtonElement>('[data-stories-ver-mas]');

if (grid) {
  // Fisher-Yates shuffle — same pattern as filter-chips.ts
  const cards = [...grid.querySelectorAll<HTMLElement>('[data-story-card]')];
  for (let i = cards.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [cards[i], cards[j]] = [cards[j], cards[i]];
  }
  cards.forEach(card => grid.appendChild(card)); // re-append in shuffled order

  let shown = Math.min(PAGE_SIZE, cards.length);

  function applyVisibility() {
    cards.forEach((card, i) => { card.hidden = i >= shown; });
    if (button) button.hidden = shown >= cards.length;
  }

  button?.addEventListener('click', () => {
    shown = Math.min(shown + PAGE_SIZE, cards.length);
    applyVisibility();
  });

  applyVisibility();
  initSharedGalleries(grid); // inits all 18 galleries (hidden ones included; safe to call once)
}

initSharedGalleryLightbox();
```

**Shuffle note:** `cards` is shuffled in place as an array first, then all cards are re-appended to the grid in the new order. This is identical to the adoption `filter-chips.ts` approach. `initSharedGalleries` is called once after the shuffle — the `data-gallery-bound` guard inside it ensures hidden galleries are initialized once (slides and event listeners) and won't be re-initialized on subsequent "Ver más" clicks.

## 8. `src/content.config.ts` Update

Add `successDogs` alongside the existing `adoptionDogs` collection:

```typescript
import { defineCollection, z } from 'astro:content';
import { glob } from 'astro/loaders';

const adoptionDogs = defineCollection({
  loader: glob({ pattern: '**/*.md', base: './src/content/adoption-dogs' }),
  schema: ({ image }) =>
    z.object({
      name: z.string(),
      sex: z.enum(['Macho', 'Hembra']),
      age: z.string(),
      weight: z.string(),
      details: z.string(),
      order: z.number().int().optional(),
      gallery: z.array(image()),
    }),
});

const successDogs = defineCollection({
  loader: glob({ pattern: '**/*.md', base: './src/content/success-dogs' }),
  schema: ({ image }) =>
    z.object({
      name: z.string(),
      story: z.string(),
      gallery: z.array(image()),
    }),
});

export const collections = {
  'adoption-dogs': adoptionDogs,
  'success-dogs': successDogs,
};
```
